# Objektově orientované programování v PHP
OOP (Object-Oriented Programming) je programovací paradigma založené na objektech.

Význam OOP v PHP:
- umožňuje strukturovat větší aplikace
- odděluje logiku do tříd
- zlepšuje údržbu kódu

Výhody:
- přehlednost
- znovupoužitelnost
- modularita

Možnosti využití OOP v PHP:
- webové aplikace (MVC architektura)
- práce s databází (modely)
- API
- systémy pro správu obsahu (CMS)

# Základní principy OOP
## Zapouzdření (encapsulation):
Zapouzdření znamená skrytí vnitřního stavu objektu a přístup k němu pouze přes definované rozhraní (metody). Data (atributy) nejsou přímo přístupné, ale pracuje se s nimi přes metody (gettery/settery).

Výhody:
- ochrana dat
- kontrola nad změnami
- snížení chyb

### Viditelnost (Visibility)
- public – viditelný odkudkoliv (jakékoliv části programu)
- private – viditelný jen ve třídě
- protected – viditelný třída + potomci

Příklad:
Data jsou skrytá (*private*) a přistupuje se k nim přes metody.
```php
<?php
class Student {
    private $vek;

    public function getVek() {
        return $this->vek;
    }

    public function setVek($vek) {
        if ($vek > 0) {
            $this->vek = $vek;
        }
    }
}

$student = new Student();
$student->setVek(20);
echo $student->getVek();
?>
```
## dědičnost (inheritance)
Dědičnost umožňuje vytvořit novou třídu na základě existující – **přebírá její vlastnosti a metody**. Používá se pro znovupoužití kódu a vytváření hierarchií tříd.

Výhody:
- znovupoužití kódu
- lepší organizace
- logická struktura

Příklad:
Třída může dědit vlastnosti a metody z jiné třídy (dědí proměnnou jmeno).
```php
<?php
class Osoba {
    public $jmeno;
}

class Student extends Osoba {
    public $rocnik;
}

$student = new Student();
$student->jmeno = "Jan";
$student->rocnik = 2;

echo $student->jmeno . " - " . $student->rocnik;
?>
```
## polymorfismus
Polymorfismus umožňuje, aby různé objekty reagovaly na stejnou metodu různě. Stejné jméno funkce → různé chování (např. podle typu objektu).

Typy:
- Překrytí (overriding) – potomkovská třída změní chování metody
- Přetížení (overloading) – stejné jméno metody, jiné parametry (**!** PHP nemá klasický overloading jako jiné jazyky, ale lze ho simulovat pomocí __call)

Výhody:
- flexibilita
- rozšiřitelnost
- čistší kód

Příklad:
```php
<?php
class Zvire {
    public function zvuk() {
        echo "Nějaký zvuk";
    }
}

class Pes extends Zvire {
    public function zvuk() {
        echo "Haf";
    }
}

class Kocka extends Zvire {
    public function zvuk() {
        echo "Mňau";
    }
}

$zvirata = [new Pes(), new Kocka()];

foreach ($zvirata as $zvire) {
    $zvire->zvuk();
    echo "<br>";
}
?>
```
# Základní prvky OOP
## Třída a objekt
**Třída** je šablona/vzor, podle kterého se vytvářejí objekty. Definuje **vlastnosti(atributy) a chování(metody)**. **K ne-statickým metodám se přistupuje pomocí** `->`.
**Objekt** je konkrétní vytvořená instance třídy.
```php
// třída
class Auto {  
	// atribut
	public $barva;

	// metoda
	public function jed() {
        echo "Auto jede";
    }
}
// objekty
$auto = new Auto();
$auto1 = new Auto();
$auto1->znacka = "Škoda";

$auto2 = new Auto();
$auto2->znacka = "BMW";

echo $auto1->znacka;
echo $auto2->znacka;
```


## Statické metody
Statické metody patří třídě a ne objektu. Nemusíme vytvářet objekt třídy aby jsme je použili. Statické vlastnosti sdílí všechny instance třídy. **Ke statickým metodám se přistupuje pomocí** `::`.
```php
class Test {
    public static $pocet = 0;

    public static function pridej() {
        self::$pocet++;
    }
}

Test::pridej();
echo Test::$pocet;
```
## Speciální metody
Speciální metody volané automaticky, **začínají** `__`:
- __construct() – při vytvoření objektu (Konstruktor)
- __destruct() – při zániku objektu
- __get() – čtení nepřístupné vlastnosti
- __set() – zápis nepřístupné vlastnosti
- __toString() – převod objektu na řetězec
- __call() – volání neexistující metody

## Výjimky (Exceptions)
Ošetření chyb oddělené od hlavní logiky pomocí `try{...}catch{...}` (Chyby jako např. chybná SQL syntaxe, chyba v připojení k databázi, ... ).
```php
try {
    throw new Exception("Chyba");
} catch (Exception $e) {
    echo $e->getMessage();
}
```

## Knihovny třetích stran
Hotový kód pro rozšíření funkčnosti aplikace.
Použití:
- instalace pomocí nástroje Composer (Správce balíčků pro PHP)
- automatické načítání tříd (autoload)

Příklady:
- PHPMailer (emaily)
- mPDF (PDF soubory)

## Frameworky
Sada nástrojů, knihoven a struktur, která ti usnadňuje psaní webových aplikací.
Základ pro vývoj aplikací (často MVC).

Výhody frameworků:
- rychlejší vývoj
- hotové řešení (routing, databáze, autentizace)
- bezpečnost
- dodržování architektury (např. MVC)

Příklady:
- Laravel
- Symfony
