# MVC
Vzor architektury webové aplikace.

MVC znamená:
- **Model** (data a logika)
- **View** (zobrazení)
- **Controller** (řízení aplikace)

Cíl architektury:
- **oddělení logiky od vzhledu** 
- **zlepšení přehlednosti kódu**
- **znovupoužitelnost částí aplikace**

Výhody:
- přehlednost
- snadná údržba
- oddělení odpovědností

Nevýhody:
- složitější návrh
- více souborů
- pro malé projekty zbytečně komplexní

### Jádro MVC
Jádro MVC tvoří:
- Router (směrovač)
- Kontrolery
- Modely
- Pohledy

#### Princip fungování
1. uživatel zadá URL
2. router určí kontroler
3. kontroler zavolá model
4. model vrátí data
5. kontroler předá data do view
6. view zobrazí výsledek

### Předávání dat mezi částmi MVC
- Controller → Model: předává parametry (např. ID uživatele)
- Model → Controller: vrací data (např. pole, objekt)
- Controller → View: předává data přes pole (např. $this->data)
- View: data pouze zobrazuje (nemění logiku)

### Celková struktura MVC
```
kontrolery/
modely/
pohledy/
css/
js/
.htaccess
index.php
init.php
```
## .htaccess
Soubor **pro konfiguraci Apache** serveru. Používá se pro **přepis URL adres(URL rewriting), skrytí index.php v adrese, směrování požadavků na jeden vstupní bod**.

Příklad **URL rewriting**:
`/index.php?url=uzivatel/2 -> /uzivatel/5`

## index.php
Takzvaný *Front Controller* - zpracovává všechny požadavky.

Zajišťuje:
- základní inicializaci aplikace
- aktivaci směrování požadavků na příslušné kontrolery
- zařazení celkového (obalujícího) pohledu, který bude v sobě zahrnovat také dílčí pohled příslušného kontroleru, do odpovědi aplikace

Příklad index.php souboru:
```php
<?php
require "init.php";

$smerovac = new SmerovacKontroler(); // nový směrovač
$smerovac->zpracuj([$_SERVER["REQUEST_URI"]]); // směrovač zpracuje URI adresu
$smerovac->vypisPohled(); // vypíše daný pohled
```

## init.php
Zajišťuje inicializační činnosti v aplikaci:
- aktivaci session
- definici a registraci autoload funkce (pro automatické načítání tříd)
- připojení k databázovému serveru

Příklad init.php souboru:
```php
<?php
function nactiTridu($nazevTridy) {
    if (preg_match("/Kontroler$/", $nazevTridy)) 
        require "kontrolery/$nazevTridy.php";
    else
        require "modely/$nazevTridy.php";
}

spl_autoload_register("nactiTridu");
```

### Autoloading (spl_autoload_register)
Autoload slouží k automatickému načítání tříd, nemusí se u každé třídy psát `require` nebo `include`.
- registruje funkci, která se zavolá při použití neznámé třídy
- PHP jí předá název třídy
- funkce podle názvu najde a načte odpovídající soubor

Výhody autoloadu:
- nemusíme ručně includovat soubory
- přehlednější kód
- lepší organizace projektu
- možnost registrovat více autoload funkcí

Příklad:
```php
function nactiTridu($nazevTridy) {
    if (preg_match("/Kontroler$/", $nazevTridy)) 
        require "kontrolery/$nazevTridy.php";
    else
        require "modely/$nazevTridy.php";
}

spl_autoload_register("nactiTridu");
```
1. vytvoříme instanci třídy (např. new UzivatelModel())
2. když PHP třídu nezná zavolá autoload funkci, která načte soubor s třídou
    (pokud název třídy končí na Kontroler, načte se ze složky kontrolery jinak ze složky model)

## Model
Datová entita, s níž v aplikaci pracujeme (ideálně třída) a která má své vlastnosti a metody (logiku práce s daty) 

Např:
 - bod
 - uživatel
 - faktura
 - článek
 - přihláška
 - objednávka
 - správce fakturace
 - správce emailů
 - databázový wrapper

### Funkce:
- načítání a ukládání dat
- validace
- výpočty

```PHP
class Uzivatel {
    private $db;

    public function __construct($db) {
        $this->db = $db;
    }

    public function najdiPodleId($id) {
        return $this->db->selectOne("SELECT * FROM uzivatele WHERE id = ?", [$id]);
    }

    public function uloz($data) {
        return $this->db->insert("uzivatele", $data);
    }
}
```

### Práce s databází
Komunikace s databází(např. MySQL, MariaDB, ...), používáme **databázový wrapper**.
**Databázový wrapper** používáme pro zjednodušení práce s DB (různé operace jako metody), pro ochranu proti *SQL injection*, a pro sjednocení operace.

Operace:
- SELECT
- INSERT
- UPDATE
- DELETE

#### Příklad Databázového wrapperu
```PHP
class Database {
    private $pdo;

    public function __construct($dsn, $user, $password) {
        $this->pdo = new PDO($dsn, $user, $password);
    }

    public function selectOne($sql, $params = []) {
        $stmt = $this->pdo->prepare($sql);
        $stmt->execute($params);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }

    public function insert($table, $data) {
        $keys = implode(",", array_keys($data));
        $values = implode(",", array_fill(0, count($data), "?"));

        $sql = "INSERT INTO $table ($keys) VALUES ($values)";
        $stmt = $this->pdo->prepare($sql);
        return $stmt->execute(array_values($data));
    }
}
```

## View
HTML šablona (s dynamicky vloženými daty), která popisuje uživatelské rozhraní a zobrazuje uživateli příslušná data a ovládací prvky. Ve view by neměla být logika, jen jednoduché výpisy.

Obsahují:
- HTML
- CSS
- JavaScript
- jednoduché PHP pro výpis dat

### Dílčí Views
Opakující se části ve všech pohledech.
Např:
- header
- nav
- footer

Pokud některé stránky se mění (např. jiná navigace, atd.), můžou se u každého pohledu přidávat samostatně:
```php
<?php include "rozlozeni.phtml"; ?>
```

Pokud ale všechny stránky používají úplně stejné části, může být v "opakujících částech" obaleno:
```html
<!DOCTYPE html>
<html lang="cs">
<head>
    <base href="/">
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>MVC app I3 WEA</title>
</head>
<body>
    <nav>
        
    </nav>
    <?php
        $this->kontroler->vypisPohled();
    ?>
</body>
</html>
```

## Controller
Prvek aplikace, který reaguje na požadavek klienta (ideálně třída). Využívá metody modelů a nakonec zajistí zaslání odpovědi klientovi s využitím vybraného (svého) pohledu.

Funkce:
- přijímá požadavky(URL, formuláře)
- volá modely
- předává data pro pohled
- vybírají pohled

```PHP
class UzivatelKontroler extends Kontroler {

    public function zpracuj($parametry) {
        $model = new Uzivatel($this->db);

        $uzivatel = $model->najdiPodleId($parametry[0]);

        $this->data['uzivatel'] = $uzivatel;
        $this->pohled = "uzivatel";
    }
}
```

Kontrolery často dědí ze základního kontroleru pro sdílení společné logiky a odstranění duplicitního kódu.

Výhody dědičnosti:
- sdílení společných metod (např. přesměrování)
- méně duplicitního kódu
- jednotná struktura kontrolerů

```PHP
abstract class Kontroler {
    protected $pohled = "";
    abstract public function zpracuj($parametry);

    public function vypisPohled() {
        require "pohledy/{$this->pohled}.phtml";
    }
    
    public function presmeruj($url) {
        header("Location:/$url");
        exit();
    }
}
```

## Router
Rozhoduje, který kontroler se spustí podle URL adresy (mapuje URL na kontrétní kontroler a akci).
```PHP
class SmerovacKontroler extends Kontroler {
    protected $kontroler;

    public function zpracuj($parametry) {
        $url = $this->parsujURL($_SERVER['REQUEST_URI']);
        $trida = ucfirst($url[0]) . "Kontroler";

        if (file_exists("kontrolery/$trida.php")) {
            $this->kontroler = new $trida();
            $this->kontroler->zpracuj($url);
            $this->pohled = $this->kontroler->pohled;
            $this->data = $this->kontroler->data;
        } else {
            echo "404 - stránka nenalezena";
        }
    }
    private function parsujURL($url) {
        $url = parse_url($url);
        $cesta = trim($url["path"], "/");
        return explode("/", $cesta);
    }
}```
