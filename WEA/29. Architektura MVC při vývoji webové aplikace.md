# MVC
Vzor architektury webové aplikace.
Cíl architektury je oddělení logiky od vzhledu a zlepšení přehlednosti kódu.
MVC znamená:
- Model
- View
- Controller

### Jádro MVC
Jádro MVC tvoří:
- Router
- Kontrolery
- Modely
- Pohledy

#### Princip
1. uživatel zadá URL
2. router určí kontroler
3. kontroler zavolá model
4. model vrátí data
5. kontroler předá data do view
6. view zobrazí výsledek

## Model
**Práce s daty** aplikace.
Funkce:
- načítání a ukládání dat
- validace
- výpočty

```PHP
Vlož model
```

### Práce s databází
Komunikace s databází(např. MySQL, MariaDB, ...), používáme **databázový wrapper**.
**Databázový wrapper** používáme pro zjednodušení práce s DB (různé operace jako metody) a pro ochranu proti *SQL injection*.
Operace:
- SELECT
- INSERT
- UPDATE
- DELETE

#### Příklad Databázového wrapperu
```PHP
Idk
```

## View
**Zobrazuje data** uživateli.
Obsahují:
- HTML
- CSS
- JavaScript

### Dílčí Views
Rozdělení části pohledů, které jsou v každém pohledu stejné.
Např:
- header
- nav
- footer

## Controller
Zpracovává požadavky a řídí celou aplikaci.
Obsahuje **logiku** aplikace.
- přijímá požadavky(URL, formuláře)
- volá modely
- předává data pro pohled
- vybírají pohled

```PHP
Vlož random kontroler
```

Kontrolery často dědí ze základního kontroleru.
```PHP
abstract class Kontroler {
    protected $pohled = "";
    abstract public function zpracuj($parametry);

    public function vypisPohled() {
        require "pohledy/{$this->pohled}.phtml";
    }
    
    public function presmeruj($url) {
        header("Location:/$url");
        exit();
    }
}
```
