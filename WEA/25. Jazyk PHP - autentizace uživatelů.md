# Autentizace
**2 podobné pojmy:**
- **Autentizace** - ověření identity
- **Autorizace** - oprávnění

Autentizace slouží k tomu, aby systém věděl, kdo právě používá aplikaci nebo web.
Příklad použití:
- přihlášení do systému
- administrace webu

## Princip autentizace
1. Uživatel zadá login a heslo.
2. Server zkontroluje údaje v databázi.
3. Pokud je vše správně, server vytvoří session pro udržení stavu uživatele.

## Session
Session umožňuje ukládat data mezi jednotlivými stránkami, např. informaci o tom, který uživatel je přihlášený.

Začíná funkcí:
```php
session_start();
```
### Pole $_SESSION
Do session lze ukládat libovolná data:
```php
$_SESSION["uzivatel"] = "admin";
```
## Funkce Session
### unset()
Smaže konkrétní proměnnou ze session.
```php
unset($_SESSION["uzivatel"]);
```
### session_destroy()
Zruší celou session a smaže v ní všechny data.
```php
session_destroy();
```
## Ukládání hesel
Hesla nikdy neukládáme přímo do databáze. Používá se **hashování** a často i **sůl (salt)** pro zvýšení bezpečnosti.
Například:
```php
// Uložení hesla
$heslo = "tajneheslo";
$hash = password_hash($heslo, PASSWORD_DEFAULT);

// Kontrola hesla při přihlášení
if (password_verify($heslo, $hash)) {
    echo "Heslo je správné!";
} else {
    echo "Heslo je nesprávné!";
}
```

# Využití databáze
Při autentizaci se data typicky ověřují proti databázi:
1. Uživatel zadá login a heslo.
2. Server načte záznam z databáze podle loginu.
3. Porovná hash hesla pomocí password_verify().
4. Pokud je správně, uloží do session informace o uživateli.

Příklad databázového dotazu:
```php
$stmt = $pdo->prepare("SELECT * FROM uzivatele WHERE login = ?");
$stmt->execute([$login]);
$user = $stmt->fetch();

if ($user && password_verify($heslo, $user["heslo_hash"])) {
    $_SESSION["uzivatel"] = $user["login"];
    $_SESSION["role"] = $user["role"];
}
```
