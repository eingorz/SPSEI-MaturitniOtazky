
## 1. Nastavení projektu (IntelliJ + MySQL driver)

Java sama o sobě neumí komunikovat s konkrétní databází (např. MySQL). Potřebuje tzv. **driver (ovladač)**.

Používá se:
- **mysql-connector-j-8.0.32.jar**

- Není to knihovna která implementuje funkce, je to driver, který implementuje rozhraní JDBC pro MySQL.
- Obecná knihovna pro práci s sql je `java.spl`

1. Stáhneš `.jar` soubor (MySQL Connector/J)
2. Otevřeš:
    - _File → Project Structure_
3. V sekci:
    - _Modules → Dependencies_
4. Přidáš:
    - **+ → JARs or Directories → vybereš .jar**
5. Scope necháš **Compile**

Tím zajistíš, že Java ví, _jak komunikovat s MySQL databází_.
## 2. JDBC – princip komunikace

JDBC je standardní API v Javě pro práci s databázemi.
### Jak funguje (teoreticky):

Java aplikace **nekomunikuje přímo s databází**, ale přes vrstvy:

Java aplikace → JDBC API → JDBC Driver → Databáze
- JDBC API = obecné rozhraní (stejné pro všechny DB)
- Driver = konkrétní implementace (např. MySQL)

Díky tomu můžeš změnit databázi bez velkých změn v kódu.
## 3. Připojení k databázi

Připojení se vytváří pomocí třídy:
- **DriverManager**
- vrací objekt typu **Connection**
- **Connection** je **AutoCloseable**, lze použít v try-with-resource bloku

### Databázová URL – rozbor

Příklad:
`jdbc:mysql://localhost:3306/mojedb`

```Java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class PripojeniDB {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/mojedb";
        String user = "root";
        String password = "heslo";

        try {
            Connection conn = DriverManager.getConnection(url, user, password);
            System.out.println("Připojeno k databázi!");

            conn.close();
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
```
### Rozklad:

| Část        | Význam                        |
| ----------- | ----------------------------- |
| `jdbc`      | označuje použití JDBC         |
| `mysql`     | typ databáze (driver)         |
| `localhost` | server (počítač, kde běží DB) |
| `3306`      | port (výchozí pro MySQL)      |
| `mojedb`    | název databáze                |
Obecný tvar:

`jdbc:<databaze>://<server>:<port>/<nazev_db>`
### Objekt Connection

- reprezentuje **otevřené spojení**
- umožňuje:
    - posílat SQL dotazy
    - vytvářet Statementy
    - řídit transakce
## **4. Třídy pro práci s databází**

Balíček: `java.sql`

### **Hlavní komponenty:**

### **1. Connection**
- spojení s databází
- vytváří další objekty
### **2. Statement**
- vykonává jednoduché SQL dotazy
- používá se pro:
    - statické dotazy bez parametrů

```Java
import java.sql.*;

public class SelectStatement {
    public static void main(String[] args) throws Exception {
        Connection conn = DriverManager.getConnection(
            "jdbc:mysql://localhost:3306/mojedb", "root", "heslo");

        Statement stmt = conn.createStatement();
        ResultSet rs = stmt.executeQuery("SELECT * FROM uzivatele");

        while (rs.next()) {
            String jmeno = rs.getString("jmeno");
            int vek = rs.getInt("vek");

            System.out.println(jmeno + " (" + vek + ")");
        }

        conn.close();
    }
}
```
### **3. PreparedStatement**
- rozšíření Statement
- SQL dotaz je:
    - **předkompilovaný**
    - obsahuje **parametry (?)**

Hlavní výhody:
- ochrana proti SQL injection
- vyšší výkon (opakované použití)
- přehlednější práce s daty

```Java
import java.sql.*;

public class PreparedExample {
    public static void main(String[] args) throws Exception {
        Connection conn = DriverManager.getConnection(
            "jdbc:mysql://localhost:3306/mojedb", "root", "heslo");

        String sql = "SELECT * FROM uzivatele WHERE jmeno = ? AND vek > ?";

        PreparedStatement ps = conn.prepareStatement(sql);
        ps.setString(1, "Jan");
        ps.setInt(2, 18);

        ResultSet rs = ps.executeQuery();

        while (rs.next()) {
            System.out.println(rs.getString("jmeno"));
        }

        conn.close();
    }
}
```
### **4. ResultSet**
- reprezentuje **výsledek SELECT dotazu**
- funguje jako „tabulka v paměti“

Ukazatel (cursor):
- pohybuje se po řádcích
- začíná **před prvním záznamem**

```Java
while (rs.next()) {
    int id = rs.getInt("id");
    String jmeno = rs.getString("jmeno");
    double plat = rs.getDouble("plat");

    System.out.println(id + " | " + jmeno + " | " + plat);
}
```
## **5. Přístup k výsledkům (ResultSet)**

### **Princip:**
- voláš `next()` → posune se na další řádek
- vrací `true/false`

### **Získání hodnot:**
- podle názvu sloupce nebo indexu

```Java
rs.getString("jmeno");  
rs.getInt("vek");
```
### **Důležité vlastnosti:**
- data jsou čtena postupně
- typ musí odpovídat (int, String…)
- existují i další metody:
    - `getDouble()`
    - `getDate()`
## 6. PreparedStatement

### **Co to je:**
Speciální typ SQL dotazu, který obsahuje **parametry místo hodnot**.

Příklad:
`SELECT * FROM uzivatele WHERE vek > ?`

Parametry se musí nastavovat postupně:
```Java
PreparedStatement ps = conn.prepareStatement(
    "SELECT * FROM uzivatele WHERE jmeno = ? AND vek > ?");

ps.setString(1, "Jan"); // 1. parametr
ps.setInt(2, 18);       // 2. parametr
```

### Pojmenované parametry (Named Parameters)

JDBC **je přímo nepodporuje**, ale používají se v různých knihovnách a frameworcích (např. Spring).

Místo `?` se používají názvy:

`SELECT * FROM uzivatele WHERE jmeno = :jmeno AND vek > :vek`

```Java
String sql = "SELECT id, Name, Age, TS FROM TestTable WHERE Age < :age OR id = :id";
try (NamedParamStatement stmt = new NamedParamStatement(conn, sql))
{
    stmt.setInt("age", 35);
    stmt.setInt("id", 2);
    ResultSet rs = stmt.executeQuery();
}
```

### **Výhody:**

- lepší čitelnost
- nezáleží na pořadí
- menší riziko chyby

### Jak funguje:
1. SQL se pošle databázi bez hodnot
2. databáze ho **předkompiluje**
3. hodnoty se doplní až později

### Proč se používá:

#### **1. Bezpečnost (SQL injection)**

Bez PreparedStatement:
`"SELECT * FROM users WHERE name = '" + input + "'"`

Útočník může vložit SQL kód

S PreparedStatement:
- hodnoty se **automaticky escapují**
- SQL kód nelze „rozbít“
#### **2. Výkon**
- dotaz se překládá jen jednou
- vhodné pro opakované použití
#### **3. Typová kontrola**

- používáš metody:
    - `setString()`
    - `setInt()`

### **Jak se používá (princip):**
1. vytvoření dotazu s `?`
2. nastavení parametrů
3. spuštění

## 7. DDL (Data Definition Language)

Slouží ke **struktuře databáze**

### **Příkazy:**
- `CREATE` – vytvoření tabulky/databáze
- `ALTER` – změna struktury
- `DROP` – smazání
- `TRUNCATE` – vymazání dat (rychlé)
### **Charakteristika:**
- mění strukturu, ne data
- většinou **nevrací ResultSet**

## 8. DML (Data Manipulation Language)

Práce s daty
### **Příkazy:**
- `SELECT` – čtení
- `INSERT` – vložení
- `UPDATE` – změna
- `DELETE` – smazání
### Charakteristika:
- pracuje s obsahem tabulek
- `SELECT` → vrací ResultSet
- ostatní → počet ovlivněných řádků

## 9. Rozdíl DDL vs DML

| DDL           | DML               |
| ------------- | ----------------- |
| struktura     | data              |
| CREATE, ALTER | SELECT, INSERT    |
| mění tabulky  | mění obsah        |
| bez ResultSet | SELECT vrací data |
