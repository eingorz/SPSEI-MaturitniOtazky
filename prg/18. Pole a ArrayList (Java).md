
# Základní principy

## Pole (array)

Datová struktura pevné velikosti
Ukládá prvky stejného typu
Paměť je alokována souvisle (kontinuálně)
Přístup k prvkům je velmi rychlý (`O(1)`)

Princip:
Pole funguje jako blok paměti, kde každý prvek má svůj index (pozici). Přístup k prvku se počítá jako:

`adresa = základ + index * velikost_prvku`

## ArrayList

Dynamická datová struktura (součást knihovny java.util)
Interně využívá pole
Velikost se automaticky zvětšuje/zmenšuje
Ukládá pouze objekty (ne primitivní typy přímo)
Pokud chceme použít primitivní datové typy, je nutno použít jejich referenční variantu (`int` -> `Integer`)

Princip:
ArrayList při zaplnění:

- Vytvoří nové větší pole (např. 2x větší)
- Zkopíruje staré prvky
- Přidá nový prvek

Operace přidání je amortizovaně `O(1)`, ale občas `O(n)` (kvůli kopírování)

# Pole v Javě

## Deklarace a inicializace
```Java
int[] pole;              // deklarace
pole = new int[5];       // vytvoření pole o velikosti 5
```

```Java
int[] pole2 = {1, 2, 3, 4}; // rovnou s hodnotami
```

## Přístup k prvkům

Syntaxe indexace
- Začíná od 0

```Java
pole[0] = 10;
System.out.println(pole[0]);
```

## Vlastnosti pole

Pevná velikost (`pole.length`)
Nelze změnit velikost po vytvoření

## Iterace pole

Cyklus s řídící proměnnou (`for`)
```Java
for (int i = 0; i < pole.length; i++) {
    System.out.println(pole[i]);
}
```

Cyklus for-each (specifický Javě)
Lze použít na jakémkoliv objektu implementující rozhraní `Iterable`
```Java
for (int x : pole) {
    System.out.println(x);
}
```

Výhody:
- Rychlý přístup
- Nízká paměťová režie

Nevýhody
Pevná velikost
Neflexibilní

# ArrayList v Javě

- Používá generika <T>
- Ukládá objekty (např. Integer místo int)

## Import
```Java
import java.util.ArrayList;
```

## Vytvoření
```Java
ArrayList<Integer> seznam = new ArrayList<>();
```

## Práce s ArrayListem

Třída `ArrayList` implementuje rozhraní `List`, které deklaruje následující metody:

### Přídání prvků (`add`,`addAll`)

```Java
list.add(prvek);        // přidá na konec
list.add(index, prvek); // vloží na index

list.addAll(jinyList);  // přidá všechny prvky
```

### Přístup k prvkům (`get`)

```Java
list.get(index);
```

### Změna prvku (`set`)

```Java
list.set(index, prvek);
```

### Odstranění prvku (`remove`,`clear`)

```Java
list.remove(index); // Vymaže index
list.remove(prvek); // Vymaže prvek

list.clear(); // Vymaže vše
```

*Poznámka: V případě že se jedná o `ArrayList<Integer>`, metoda remove se rozhoduje jestli odebere index či prvek podle datového typu `int` nebo `Integer`*

### Stav pole (`size`,`isEmpty`)

```Java
list.size(); // Velikost
list.isEmpty(); // boolean - je prázdné
```

### Vyhledávání (`contains`,`index`,`lastIndexOf`)

```Java
list.contains(prvek); // boolean - pole obsahuje prvek
list.index(prvek); // najde index prvniho vyskytu prvku
list.lastIndexOf(prvek); // najde index posledniho vyskytu prvku
```

### Iterace

Cyklus s řídící proměnnou (`for`)
```Java
for (int i = 0; i < pole.size(); i++) {
    System.out.println(pole.get(i));
}
```

Cyklus for-each (specifický Javě)
Lze použít na jakémkoliv objektu implementující rozhraní `Iterable`
```Java
for (Integer x : pole) {
    System.out.println(x);
}
```

### Pokročilejší manipulace
- `sort`
- `reverse`
- `subList`
atd.

### Převod na pole (`toArray`)

```Java
Object[] pole = list.toArray(); // Vraci vzdy array prvku "Object"
String[] pole2 = list.toArray(new String[0]); //Vraci array daneho datoveho typu
```

## Koncept kapacity ArrayListu

ArrayList má určitou kapacitu při vytvoření
Když potřebuje růst kvůli vkládání prvků, přealokuje se a navýší kapacitu na dvojnásobek

```Java
ArrayList<String> arr = new ArrayList<>();

arr.ensureCapacity(100); // rezervuje kapacitu
arr.trimToSize();        // zmenší kapacitu na aktuální velikost
```

Výhody:
- Flexibilní velikost
- Jednoduché použití

Nevýhody:
- Pomalejší než pole
- Vyšší paměťová režie
- autoboxing (`int` -> `Integer`)
