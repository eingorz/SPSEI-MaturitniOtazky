Proměnné jsou abstraktní koncepty pro reprezentování paměti a práci s ní.

## 5.1 Statické proměnné

Statická proměnná je pojmenované místo v paměti, které uchovává data na zásobníku. V jazyce C musí být každá proměnná deklarována před použitím, aby kompilátor věděl, jaký datový typ a kolik paměti pro ni alokovat. Ve standardu ANSI C tato deklarace musí být provedena na začátku části kódu, v niž je daná proměnná platná.

## Deklarace, definice a inicializace

*   **Deklarace**: Informuje kompilátor o existenci proměnné, jejím názvu a datovém typu. Nealokuje paměť
    ```c
    extern int pocet; // Deklarace, že proměnná 'pocet' typu int existuje jinde
    ```

*   **Definice**: Alokuje paměť pro proměnnou. Většinou zahrnuje i deklaraci.
    ```c
    int a; // Definice proměnné 'a' typu int
    ```

*   **Inicializace**: Přiřazení počáteční hodnoty proměnné při její definici. Pokud proměnná není inicializována, její hodnota je nedefinovaná (garbage value), pokud není globální nebo statická (ty jsou automaticky inicializovány na nulu).

    ```c
    int b; // Deklarace a definice
    b = 10; // Inicializace (přiřazení hodnoty)

    int c = 20; // Deklarace, definice a inicializace v jednom kroku
	```


```c
    // Příklad z poznámek:
    int b_globalni; // Globální proměnná
    void main(){
        int a_lokalni = 0; // Lokální proměnná, definice a inicializace
        // datový typ, identifikátor, hodnota
    }
```

*Poznámka: Statické globální proměnné jsou vždy naalokovány v sektoru jenž je před skokem do funkce `main` vynulován, statické lokální proměnné jsou ovšem na zásobníku a není možno počítat s tím, že jejích počáteční hodnota bude nulová*
## Datové typy

Základní datové typy v C:

*   **`int`**: Celé číslo. Velikost závisí na architektuře, obvykle 2 nebo 4 bajty. Rozsah je typicky od -2,147,483,648 do 2,147,483,647 (pro 32-bit).
    *   `short int` (nebo `short`): Menší celá čísla (obvykle 2 bajty).
    *   `long int` (nebo `long`): Větší celá čísla (obvykle 4 nebo 8 bajtů).
    *   `unsigned int`: Pouze nezáporná celá čísla (rozsah od 0 do cca 4,294,967,295).
*   **`char`**: Jeden znak (ASCII hodnota). Obvykle 1 bajt. Může být interpretován jako malé celé číslo.
    *   `signed char`: Od -128 do 127.
    *   `unsigned char`: Od 0 do 255.
*   **`float`**: Číslo s plovoucí desetinnou čárkou (jednoduchá přesnost). Obvykle 4 bajty. Rozsah cca ±3.4e38.
*   **`double`**: Číslo s plovoucí desetinnou čárkou (dvojitá přesnost). Obvykle 8 bajtů. Větší rozsah a přesnost než `float`.
*   **`void`**: Označuje absenci typu. Používá se pro funkce, které nevrací hodnotu, nebo pro generické ukazatele.

Příklady s různými datovými typy a s "literal" výrazy každého z nich při inicializaci statické proměnné:
```c
int vek = 30;
char pismeno = 'A';
float teplota = 25.5f; // 'f' označuje float konstantu
double pi = 3.1415926535;
```

## Scope (Rozsah platnosti)

*   **Lokální proměnné**:
    *   Deklarované uvnitř bloku kódu (např. uvnitř funkce, cyklu `for`, bloku `{}`).
    *   Jejich platnost začíná od místa deklarace a končí na konci bloku, ve kterém jsou deklarovány.
    *   Pokud lokální proměnná má stejné jméno jako globální proměnná, **lokální proměnná má přednost** (tzv. "shadowing") v rámci svého rozsahu platnosti.
*   **Globální proměnné**:
    *   Deklarované mimo všechny funkce, obvykle na začátku souboru.
    *   Jejich platnost začíná od místa deklarace a trvá do konce souboru (nebo modulu, pokud nejsou `static`).
    *   Jsou automaticky inicializovány na nulu, pokud nejsou explicitně inicializovány.

Příklad "shadowing":
```c
int x = 10; // Globální x

void funkce() {
    int x = 5; // Lokální x, "stíní" globální x
    printf("Lokální x: %d
", x); // Vypíše 5
}

int main() {
    funkce();
    printf("Globální x: %d
", x); // Vypíše 10
    return 0;
}
```

## Paměťové třídy (Storage Classes)

Určují životnost, rozsah platnosti a umístění proměnné v paměti.

*   **`auto`**:
    *   Výchozí třída pro lokální proměnné.
    *   Proměnné mají automatickou životnost – jsou vytvořeny při vstupu do bloku a zničeny při výstupu z bloku.
    *   Paměť je alokována na zásobníku (stack).
    ```c
    void priklad_auto() {
        auto int pocet = 0; // Stejné jako int pocet = 0;
        // ... pocet existuje ...
    } // pocet je zde zničen
    ```
*   **`extern`**:
    *   Používá se pro deklaraci proměnné, která je definována v jiném souboru (modulu).
    *   Říká kompilátoru, že proměnná existuje, ale její paměť bude alokována jinde.
    *   Nemá vlastní životnost, odkazuje na globální proměnnou.
    ```c
    // V souboru.c
    int globalni_promenna = 100;

    // V jinem_souboru.c
    extern int globalni_promenna; // Deklarace, že globalni_promenna je definována jinde
    void pouzij_globalni() {
        printf("%d
", globalni_promenna); // Použije globální proměnnou
    }
    ```
*   **`register`**:
    *   Indikuje kompilátoru, že proměnná by měla být uložena v registru procesoru pro rychlejší přístup.
    *   Používá se pro lokální proměnné, které jsou často používány (např. v cyklech).
    *   Počet registrů je omezený, kompilátor nemusí `register` vždy respektovat.
    *   Nelze získat adresu `register` proměnné pomocí referenčního operátoru `&`
    *   Ne všechny datové typy mohou použít paměťovou třídu `register`, protože ne všechny datové typy (např. programátorem definovaný `struct` nebo `union`) se do registrů velikostně vlezou.
    ```c
    void rychle_pocitani() {
        register int i; // Žádost o uložení 'i' do registru
        for (i = 0; i < 1000; i++) {
            // ...
        }
    }
    ```
*   **`static`**:
    *   **Pro lokální proměnné**: Dává proměnné statickou životnost. Zůstává v paměti i po opuštění bloku, kde byla deklarována, a uchovává si svou hodnotu mezi jednotlivými voláními funkce. Je inicializována pouze jednou při prvním volání funkce.
    *   **Pro globální proměnné**: Omezuje jejich rozsah platnosti pouze na daný soubor (modul). Jsou pak nepřístupné z jiných souborů, ani pomocí `extern`.
    ```c
    void pocet_volani() {
        static int pocet = 0; // Inicializováno pouze jednou na 0
        pocet++;
        printf("Funkce byla volána %d krát.
", pocet);
    }

    // V souboru.c
    static int skryta_globalni = 50; // Přístupná jen v souboru.c

    int main() {
        pocet_volani(); // Vypíše: Funkce byla volána 1 krát.
        pocet_volani(); // Vypíše: Funkce byla volána 2 krát.
        return 0;
    }
    ```

-   `volatile`
	- **Volatile** říká kompilátoru že hodnota této proměnné se může asynchronně měnit za běhu programu (ať už to v případě vestavěných systémů a nebo asynchronní použití vláken)
	
	- Nutí kompilátor před každou operací s hodnotou proměnné ji znovu přečíst v paměti a neoptimalizovat její využití tím, že si ji načte do registru pouze jednou.
## Konstanty

Konstanty jsou hodnoty, které se během běhu programu nemění.

*   **`const`**: Klíčové slovo pro definování konstantních proměnných. Kompilátor zajistí, že hodnota proměnné nebude změněna.
    ```c
    const float PI = 3.14159f;
    const int MAX_POCET = 100;
    // PI = 3.0; // Toto by způsobilo chybu kompilace
    ```
*   **Preprocesorové makro `#define`**: Jiný způsob definování konstant. Makra jsou nahrazena textově kompilátorem před samotným překladem.
    ```c
    #define UHEL_PRAVY 90
    // Před kompilací se UHEL_PRAVY nahradí číslem 90
    ```
    Rozdíl je v tom, že `const` je proměnná s typem, zatímco `#define` je jen textové nahrazení.

## Přetypování (Type Casting)

Přetypování je proces konverze hodnoty jednoho datového typu na jiný.

*   **Implicitní přetypování**: Kompilátor automaticky provede konverzi, pokud je to bezpečné (např. `int` na `float`).
    ```c
    int cislo_int = 10;
    float cislo_float = cislo_int; // cislo_float bude 10.0f
    ```
*   **Explicitní přetypování**: Programátor explicitně určí, jak má být hodnota konvertována pomocí operátoru přetypování `(typ)`.
    ```c
    float deleni = 7.0f / 2; // Výsledek bude 3.5f
    int vysledek_int = (int)deleni; // Přetypování na int, výsledek bude 3 (část za desetinnou čárkou se ztratí)

    int deleni_int = 7 / 2; // Celá čísla, výsledek bude 3
    float vysledek_float = (float)deleni_int; // Přetypování na float, vysledek_float bude 3.0f
    ```
## 5.2 Dynamické proměnné

Dynamické proměnné jsou nepojmenované části paměti na haldě ke kterým z pravidla přistupujeme pomocí pointerových statických proměnných na zásobníku.

### Funkce malloc

K vytvoření dynamické proměnné používáme funkci standardní knihovny `stdlib.h` `malloc`

```C
	void *malloc(size_t size);
```

Z definice funkce malloc vyplývá že jako vstupní parametr vyžaduje velikost paměťového bloku v bytech a vrací pointer na `void`, tedy adresu začátku tohoto bloku paměti.

Pro práci s tímto blokem paměti jako s daným datovým typem přetypováváme vrácený pointer na `void` na daný datový typ

```C
	int *ptr = (int*)malloc(sizeof(int));
```

*Poznámka V tomto příkladě ke zjištění velikosti datového typu `int` v bytech používáme operátor `sizeof`

Pomocí funkce `malloc` taktéž můžeme vytvořit pole s délkou neurčitelnou při kompilaci.

```C
	int *pole = (int*)malloc(10 * sizeof(int)); // 10 je zde velikost pole
```

Zde tvoříme blok paměti o velikosti deseti integerů a pro práci s polem můžeme využít faktu že pole je pointer na první prvek paměťového bloku stejně jako pole

### Funkce free

Ke zničení tohoto paměťového bloku v případech kde s nim již nepotřebujeme pracovat používáme funkci standartní knihovny `stdlib.h` `free`

```c
	void free(void *ptr);
```

Z definice funkce `free` vyžaduje jako jedinný parametr adresu začátku paměťového bloku.

Po ukončení práce s paměťovým blokem se častokrát hodnota statické pointerové proměnné jenž jsme využívali k přístupu k tomuto paměťovému bloku nastavuje na `NULL` tedy na nulovou adresu `((void*)0)`. Tím se zamezí, že při případné semantické chybě prací s uvolňeným blokem paměti by se přepsali data a program v tomto případě pouze spadne se segmentační chybou.
