
## Třída (class)

Třída je **šablona / předpis**, podle kterého se vytvářejí objekty.

Popisuje:
- **atributy (proměnné)** – vlastnosti
- **metody** – chování

### Příklad třídy

```Java
class Pes {  
    String jmeno;  
    int vek;  
  
    void stekej() {  
        System.out.println("Haf");  
    }  
}
```

Třída říká:
- jaké má pes vlastnosti (`jmeno`, `vek`)
- co umí dělat (`stekej()`)

## **2. Objekt (instance)**

Objekt je **konkrétní instance třídy** – reálný „kus“ vytvořený podle třídy.

Každý objekt má:
- vlastní hodnoty atributů
- stejné metody jako třída

### **Vytvoření objektu**

```Java
Pes p1 = new Pes();  
p1.jmeno = "Rex";  
p1.vek = 5;  
  
p1.stekej();
```

### **Více objektů**

```Java
Pes p1 = new Pes();  
Pes p2 = new Pes();
```

Každý objekt je **nezávislý**
## **Rozdíl mezi třídou a objektem**

| Třída               | Objekt               |
| ------------------- | -------------------- |
| šablona             | konkrétní instance   |
| popisuje vlastnosti | má konkrétní hodnoty |
| nevykonává se sama  | existuje v paměti    |
## Dědičnost (inheritance)

Dědičnost umožňuje vytvářet nové třídy na základě existujících.

Nová třída (potomek) přebírá:

- atributy (proměnné)
- metody
### **Základní pojmy**

- **nadtřída (superclass)** – rodič
- **podtřída (subclass)** – potomek

### **Syntaxe**

```Java
class Zvire {  
    void zvuk() {  
        System.out.println("Nějaký zvuk");  
    }  
}  
  
class Pes extends Zvire {  
    void stekej() {  
        System.out.println("Haf");  
    }  
}
```

`extends` = dědičnost

### **Přepisování metod (override)**

Podtřída může změnit chování metody:

```Java
@Override  
void zvuk() {  
    System.out.println("Haf");  
}
```
## **2. Dědičnost vs. kompozice**

### **Dědičnost = „JE“ (IS-A)**

`Pes JE Zvire`

Používá se, když:
- existuje přirozený vztah
- chceme sdílet chování

### **Kompozice = „MÁ“ (HAS-A)**

Třída obsahuje jinou třídu jako atribut:

```Java
class Motor {  
}  
  
class Auto {  
    Motor motor; // Auto má motor  
}
```

### **Shrnutí**

| Dědičnost   | Kompozice      |
| ----------- | -------------- |
| JE          | MÁ             |
| extends     | objekt uvnitř  |
| silná vazba | volnější vazba |
V praxi se často preferuje **kompozice (větší flexibilita)**
## **3. Konstruktory v dědičnosti**

Konstruktor podtřídy volá konstruktor nadtřídy pomocí:

```Java
super();
```
### **Příklad**

```Java
class Zvire {  
    Zvire() {  
        System.out.println("Vytvořeno zvíře");  
    }  
}  
  
class Pes extends Zvire {  
    Pes() {  
        super(); // volání konstruktoru rodiče  
        System.out.println("Vytvořen pes");  
    }  
}
```

`super()` se volá automaticky (pokud není uvedeno)

### **Parametrický konstruktor**

```Java
class Zvire {  
    String jmeno;  
  
    Zvire(String jmeno) {  
        this.jmeno = jmeno;  
    }  
}  
  
class Pes extends Zvire {  
    Pes(String jmeno) {  
        super(jmeno); // MUSÍ být zavoláno  
    }  
}
```
## **4. Přístupová práva a dědičnost**

### **Modifikátory přístupu**

| Modifikátor | Přístup           |
| ----------- | ----------------- |
| `private`   | jen v té třídě    |
| `default`   | v balíčku         |
| `protected` | balíček + potomci |
| `public`    | všude             |

### **Chování v dědičnosti**

- `private` → nejsou přístupné
- `protected` → dostupné v potomcích
- `public` → dostupné všude
### **Příklad**

```Java
class A {  
    private int x;  
    protected int y;  
}  
  
class B extends A {  
    void test() {  
        System.out.println(y); // x nelze  
        System.out.println(y); // y lze  
    }  
}
```

## **5. Klíčové slovo final**

`final` znamená **nelze změnit / přepsat / zdědit**

### **1. final proměnná**
```Java
final int CISLO = 10;
```

Nelze změnit (konstanta)
### **2. final metoda**
```Java
final void metoda() {  
}
```

Nelze přepsat v podtřídě
### **3. final třída**
```Java
final class A {  
}
```

Nelze dědit

(Příklad z praxe: `String` je final)
