# Obecně
- Datové typy určují rozsah hodnot, jaké můžeme provádět operace a jak jsou interpretovány data
- **Statické typování:** C i Java jsou staticky typované jazyky. Datový typ proměnné musí být pevně určen už při kompilaci a nelze jej za běhu programu změnit.

### Definice struct a union (s typedef)
Abychom nemuseli při každém vytváření proměnné v C psát klíčové slovo `struct` nebo `union`, definují se tyto strukturované typy nejčastěji rovnou pomocí `typedef`.

```c
// STRUCT
// Normální definice
struct Osoba {
    char jmeno[50];
    int vek;
};
struct Osoba clovek1; // Při deklaraci musíme psát slovo "struct"

// Definice s okamžitým vytvořením instancí
struct Auto {
    char znacka[50];
    int rokVyroby;
} auto1, auto2; // Proměnné auto1 a auto2 jsou vytvořeny rovnou s definicí struktury

// Definice s typedef 
typedef struct {
    char jmeno[50];
    int vek;
    float vyska;
} Student;

Student s1; // Rovnou používáme "Student" jako nový datový typ
s1.vek = 20; // Přístup k prvkům přes tečkovou notaci

// Přístup přes pointer
Student *ptr = &s1;
ptr->vek = 21; // U pointerů na struct se používá šipka (->)


// --- UNION ---
// Definice s typedef
typedef union {
    int celeCislo;
    float desetinneCislo;
    char znak;
} Hodnota;

Hodnota h1;
// U unionu všechny prvky sdílí stejnou paměť! 
// Velikost unionu = velikost největšího prvku (zde int/float zabírají 4 byty)
h1.celeCislo = 42;
h1.desetinneCislo = 3.14; // POZOR: Tímto se přepíše hodnota celeCislo!
```

### typedef
- Vytváří alias (nové jméno) pro existující datový typ
- Zjednodušuje zápis komplexnějších typů a struktur
```c
typedef struct {
    int x;
    int y;
} Bod;

Bod b1; // Místo "struct Bod b1;"
```

### String (Textový řetězec)
V céčku neexistuje nativní datový typ pro text. String se realizuje jako **pole znaků** (`char`), které je vždy ukončeno nulovým znakem `\0` (null-terminator). Více viz [[14. Znaky a řetězce (C)]].

### Enum (Výčtový typ)
Uživatelsky definovaný typ, který přiřazuje jména celočíselným konstantám (defaultně od 0).
```c
typedef enum {
    PONDELI,  // 0
    UTERY,    // 1
    STREDA    // 2
} Dny;

Dny dnes = PONDELI;
```

### Boolean (Logická hodnota)
Nativně v původním C chybí (používá se `0` pro false a nenulová hodnota pro true). Od standardu C99 lze po inkluzi `<stdbool.h>` používat typ `bool` a hodnoty `true` / `false`.

### Paměťové třídy
Určují životnost, rozsah platnosti a umístění proměnné v paměti.
Viz. [[5. Proměnné v jazyce C]].

## Java
### Srovnání: Primitivní vs. Referenční typy

| Vlastnost       | Primitivní typy (Primitive)     | Referenční typy (Reference)                    |
| :-------------- | :------------------------------ | :--------------------------------------------- |
| **Definice**    | 8 základních, pevně daných typů | Mohou být uživatelsky definované (třídy, pole) |
| **Co ukládají** | Ukládají přímo **samotná data** | Ukládají **adresu** v paměti (odkaz na objekt) |
| **Obsah**       | Pouze jedna konkrétní hodnota   | Mohou obsahovat více proměnných a metod        |
| **Paměť**       | Zabírají méně paměti            | Jsou paměťově náročnější                       |
| **Rychlost**    | Rychlejší zpracování            | Pomalejší (přístup přes referenci)             |

### Primitivní datové typy (8 základních)

Nejsou to objekty, nemají metody. Ukládají se na zásobníku (Stack).
- **Celočíselné:**
  - `byte` (8 bitů / 1 byte)
  - `short` (16 bitů / 2 byty)
  - `int` (32 bitů / 4 byty) - *nejpoužívanější*
  - `long` (64 bitů / 8 bytů) - *píše se s příponou L, např. 100L*
- **Desetinné:**
  - `float` (32 bitů / 4 byty) - *píše se s příponou f, např. 3.14f*
  - `double` (64 bitů / 8 bytů) - *výchozí pro desetinná čísla*
- **Znakové:**
  - `char` (16 bitů / 2 byty) - *obsahuje jeden znak v kódování Unicode, píše se do apostrofů 'a'*
- **Logické:**
  - `boolean` (1 byte) - *nabývá hodnot `true` nebo `false`*
### Referenční (objektové) datové typy

Ukládají odkaz (adresu) na místo v paměti (Heap - halda), kde je uložen samotný objekt. Na Stacku je uložena pouze tato adresa.
- **Třídy (Classes) a Rozhraní (Interfaces):** Uživatelsky definované datové typy.
- **String:** Textový řetězec. V Javě je to plnohodnotný objekt, nikoliv jen pole znaků jako v C. Je **neměnný (immutable)**.
- **Pole (Arrays):** Např. `int[] cisla = new int[5];`. V Javě je pole vždy objekt, a to i když obsahuje primitivní typy.
### Obalovací třídy (Wrapper classes)

Ke každému primitivnímu typu existuje jeho objektová (referenční) varianta (např. `int` -> `Integer`, `double` -> `Double`, `char` -> `Character`, `boolean` -> `Boolean`).
- Používají se tam, kde jsou vyžadovány objekty (např. v kolekcích jako `ArrayList<Integer>`, protože `ArrayList<int>` nelze vytvořit).
- **Autoboxing / Unboxing:** Java umí automaticky převádět primitivní typy na obalovací třídy a naopak.
### Výchozí hodnoty (Default values) a null
Pokud jsou proměnné deklarovány jako atributy třídy (globálně, nikoli lokálně v metodě) a nejsou inicializovány:
- **Primitivní typy:** Mají výchozí hodnoty (`0` pro čísla, `false` pro boolean).
- **Referenční typy:** Mají výchozí hodnotu `null` (odkazují "nikam"). Primitivní typ nikdy nemůže být `null`.
