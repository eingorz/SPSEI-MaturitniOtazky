### 6.1. Co je projekt v jazyce C

Jako **Projekt** označujeme dva a více zdrojových souborů, hlavičkových souborů a dalších součástí, které dohromady tvoří jeden program.  

U větších programů se kód rozděluje do více souborů, aby byl:
- přehlednější
- snadněji udržovatelný
- znovupoužitelný

Projekt obvykle obsahuje:
- zdrojové soubory (`.c`)
- hlavičkové soubory (`.h`)
- případně objektové soubory knihoven

Při kompilaci **kompilátor** všechny zdrojové soubory přeloží samostatně a následně **linker** spojí výsledné objektové soubory do jednoho spustitelného programu.

---
## 6.2 Zdrojové soubory (.c)

**Zdrojový soubor** obsahuje samotnou implementaci programu.

Typicky obsahuje:
- definice funkcí
- definice globálních proměnných
- jeden z nich obsahuje hlavní funkci `main()`

Příklad zdrojového souboru `main.c`:

```c
#include <stdio.h>  
#include "matematika.h"  
  
int main() {  
    int vysledek = soucet(3, 4);  
    printf("Vysledek: %d\n", vysledek);  
    return 0;  
}
```

Další zdrojový soubor `matematika.c`:
```c
#include "matematika.h"  
  
int soucet(int a, int b) {  
    return a + b;  
}
```
Každý `.c` soubor se kompiluje **samostatně** do objektového souboru (`.o`).

---
## 3. Hlavičkové soubory (.h)

**Hlavičkový soubor** obsahuje především **deklarace**, nikoliv implementaci.

Typicky obsahuje:
- deklarace funkcí
- vlastní datové typy (`typedef`)
- deklarace struktur, enumů, atd.
- konstanty (`#define`)
- deklarace externích proměnných (`extern`)

Hlavičkový soubor `matematika.h`:
```c
#ifndef MATEMATIKA_H  
#define MATEMATIKA_H  
  
int soucet(int a, int b);  
  
#endif
```

Tento soubor se pak připojuje pomocí preprocesorové direktivy:

`#include "matematika.h"`
## 4. Rozdíl mezi deklarací a definicí

### Deklarace
Informuje překladač o existenci funkce nebo proměnné.

`int soucet(int a, int b);`

### Definice
Obsahuje samotnou implementaci.

```c
int soucet(int a, int b) { 
    return a + b;  
}
```

Deklarace bývá v **.h souboru**, definice v **.c souboru**.
## 5. Direktiva `#include`

Slouží k vložení obsahu hlavičkového souboru do programu.
Existují dva typy:

### Systémové knihovny

`#include <stdio.h>`

- Označené pomocí `<>`, preprocesor tyto soubory většinou hledá ve všech cestách, které podle něj byly označené za knihovnové složky 
- u gcc parametr `-I` pro přidání této cesty
	`gcc -I/path/to/headers`)

Používají se pro standardní knihovny jazyka C.
### Vlastní hlavičkové soubory

`#include "matematika.h"`

- Preprocesor hledá soubory relativně k právě procházenému souboru.
- Používají se pro soubory v projektu.
## 6. Ochrana proti vícenásobnému zahrnutí (Include Guard)

Pokud by byl hlavičkový soubor načten vícekrát, vznikly by chyby při kompilaci.

Proto se používají tzv. **include guards** pomocí podmíněného překladu:

```c
#ifndef MATEMATIKA_H  
#define MATEMATIKA_H  
  
int soucet(int a, int b);  
  
#endif
```

Princip:

1. pokud symbol není definován (`#ifndef`)
2. definuj ho (`#define`)
3. vlož obsah souboru
4. při dalším načtení se již obsah nevloží

## 7. Proces překladu projektu

Překlad programu v C probíhá v několika krocích:

-  Preprocesor
	Zpracuje direktivy:
-  Kompilace
	Každý `.c` soubor se přeloží do **objektového souboru** (`.o`).
-  Linkování
	Linker spojí všechny objektové soubory a knihovny do **spustitelného souboru**.

Výsledkem je většinou spustitelný program
## 8. Důvody rozdělení programu do více souborů

- **přehlednost** – menší soubory jsou čitelnější
- **modularita** – každá část programu má vlastní soubor
- **znovupoužitelnost** – stejné funkce lze použít v jiných projektech
- **rychlejší kompilace** – překládají se jen změněné soubory (make)
- **týmová práce** – více programátorů může pracovat současně
