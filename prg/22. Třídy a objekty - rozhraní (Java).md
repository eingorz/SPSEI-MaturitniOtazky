
## 1. Co je rozhraní (interface)

Rozhraní je kostra/předpis pro to, co má třída implementující rozhraní definovat.

Rozhraní obsahuje:
- deklarace metod (bez implementace)
- konstanty

Určuje **CO má třída dělat, ale ne JAK**
### **Syntaxe**
```Java
interface Zvire {  
    void zvuk();  
}
```
## 2. Implementace rozhraní

Třída, která rozhraní implementuje, musí **definovat všechny jeho metody**.

```Java
class Pes implements Zvire {  
    @Override  
    public void zvuk() {  
        System.out.println("Haf");  
    }  
}
```

`implements` = implementace rozhraní
## 3. Vlastnosti rozhraní
- metody jsou implicitně:
    - `public`
    - `abstract`
- proměnné jsou:
    - `public static final` (konstanty)
### **Příklad**

```Java
interface Test {  
    int MAX = 100; // konstanta  
  
    void metoda(); // public abstract  
}
```

## **4. Vícenásobná implementace**

Java nepodporuje vícenásobnou dědičnost tříd, ale:

**Třída může implementovat více rozhraní**

```Java
interface A {  
    void a();  
}  
  
interface B {  
    void b();  
}  
  
class C implements A, B {  
    public void a() {}  
    public void b() {}  
}
```
## **5. Default a static metody (Java 8+)**

### **Default metoda**
- má implementaci
- není nutné ji přepisovat

```Java
interface A {  
    default void pozdrav() {  
        System.out.println("Ahoj");  
    }  
}
```
### **Static metoda**

```Java
interface A {  
    static void info() {  
        System.out.println("Info");  
    }  
}
```
## 6. Rozhraní vs. abstraktní třída

| Rozhraní                 | Abstraktní třída        |
| ------------------------ | ----------------------- |
| jen metody (většinou)    | může mít i implementaci |
| vícenásobná implementace | jen jedna dědičnost     |
| `implements`             | `extends`               |

## **7. Polymorfismus s rozhraním**

Proměnná může být typu rozhraní:

```Java
Zvire z = new Pes();  
z.zvuk();
```

Umožňuje obecnější a flexibilní kód

## **8. Výhody rozhraní**

- oddělení **definice od implementace**
- podpora polymorfismu
- umožňuje vícenásobné chování
- lepší návrh programu

## **9. Funkční rozhraní (Java 8+)**

Rozhraní s **jednou metodou**

```Java
@FunctionalInterface  
interface A {  
    void metoda();  
}
```

Používá se pro:
- lambda výrazy

## **Shrnutí**

- `interface` = smlouva (co má třída umět)
- `implements` = implementace
- metody jsou `public abstract`
- proměnné jsou konstanty
- lze implementovat více rozhraní
- podporuje polymorfismus
