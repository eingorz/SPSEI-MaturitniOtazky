## Základní princip  

- Proměnná datového typu pointerového uchovává **adresu v paměti**.  
- Typ pointeru určuje **jaký typ dat se na dané adrese očekává**.  
- Pointer umožňuje:
	- přímou manipulaci s pamětí 
	- efektivní předávání argumentů funkcím
	- implementaci dynamických datových struktur (listy, stromy, grafy)

```c  
int x = 10;  
int *p = &x; // p obsahuje adresu proměnné x
```
## Referenční a dereferenční operátor

### Referenční operátor `&`

- vrací **adresu proměnné v paměti**
- používá se při inicializaci pointeru

```c
int x = 5;  
int *p = &x;
```
### Dereferenční operátor `*`

- přistupuje k **hodnotě uložené na adrese**, kterou pointer obsahuje
```c
int x = 5;  
int *p = &x;  
  
printf("%d\n", *p); // vypíše 5
```

Změna hodnoty přes pointer:
```c
*p = 20;   // změní hodnotu x na 20
```

## Deklarace pointerů

Syntaxe:
```c
typ *jméno;
```

příklady:
```c
int *p;  
double *dp;  
char *cp;
```

Více pointerů:
```c
int *p1, *p2;
```

Pozor:
```c
int* a, b;  // pouze a je pointer
```

## Null pointer
- pointer může obsahovat **nulovou adresu**
- označuje **neplatný nebo neinitializovaný pointer**
- makro `NULL` je definováno ve více standartních knihovnách (`stdio.h`, `stdlib.h`, `stddef.h`), vždy je rozšířeno na `((void*)0)`

```c
int *p = NULL;
```

Používá se pro:
- kontrolu platnosti pointeru
- ukončení struktur (např. spojové seznamy)

Principem je že dereference nulového pointeru my měla vyústit v segmentační chybu

## Pointer a pole

V jazyce C je **pole v mnoha kontextech interpretováno jako pointer na první prvek**.

```c
int arr[5] = {1,2,3,4,5};  
  
int *p = arr;      // ekvivalentní: &arr[0]
```

Platí:
```c
arr[i] == *(arr + i)
```

Příklad:
```c

printf("%d\n", *(arr + 2)); // 3
```

Důležité rozdíly:
- `arr` je **konstantní adresa** (nelze měnit)
- `p` je **proměnný pointer**

```c
p = p + 1;   // OK  
arr = arr+1; // chyba
```

## Pointerová aritmetika

Pointery podporují **aritmetické operace**, které respektují velikost typu.
Operace:
- `p + n`
- `p - n`
- `p1 - p2`
- `p++`, `p--`

Posun pointeru:

```c
int arr[5] = {1,2,3,4,5};  
int *p = arr;  
  
p++;  // posun o sizeof(int)
```

Interně:
```c
p = p + 1  // adresa + sizeof(datatype)
```

Příklad:
```c
for(int *p = arr; p < arr + 5; p++){  
    printf("%d\n", *p);  
}
```

Rozdíl pointerů:
```c
int diff = p2 - p1;
```

výsledek = **počet prvků mezi adresami**, nikoliv počet bytů.

## Pointery jako parametry funkcí

Používají se pro **simulaci předávání referencí**.

```c
void swap(int *a, int *b) {  
    int tmp = *a;  
    *a = *b;  
    *b = tmp;  
}
```

Použití:

```c
int x = 5;  
int y = 10;  
  
swap(&x, &y);
```

## Pointer na pointer

Pointer může ukazovat na další pointer.
```c
int x = 5;  
int *p = &x;  
int **pp = &p;
```

Dereference:
```c
printf("%d\n", **pp);
```

Používá se například u:
- dynamických 2D polí
- práce s argumenty funkcí
- manipulace s pointery ve funkcích

---
## Pointer a dynamická paměť

Dynamická alokace pomocí `malloc`, `calloc`, `realloc`.
```c
int *arr = malloc(10 * sizeof(int));
```

Alokuje blok paměti na haldě, pokud není možno paměť alokovat, vrací `NULL`

Uvolnění:
```c
free(arr);
```

Bez uvolnění vzniká **memory leak** (semantická chyba úniku paměti).

---
## Pointery a struktury
Pointer na strukturu:
```C
struct Point {  
    int x;  
    int y;  
};  
  
struct Point p = {1,2};  
struct Point *ptr = &p;
```

Přístup:
```c
ptr->x  
ptr->y
```

Ekvivalent:
```c
(*ptr).x
```

## Klíčové slovo `register`

`register int x`;

- doporučuje kompilátoru uložit proměnnou do **CPU registru**
- proměnná **nemá garantovanou paměťovou adresu**
- proto **nelze použít operátor `&`**

```c
register int x = 5;
  
int *p = &x; // chyba
```

Dnes má význam spíše historický (moderní kompilátory optimalizují samy).

---
## Typické chyby při práci s pointery

### Dereference neinitializovaného pointeru

```c
int *p;  
*p = 5; // undefined behavior
```

Pokud je pointer globální, globální blok paměti je vynulován a jedná se tak o nulový pointer, jeho dereferencí je způsobena na většině operačních systémů segmentační chyba. 

---
### Dangling pointer

Pointer ukazuje na již uvolněnou paměť.
```c
int *p = malloc(sizeof(int));  
free(p);  
  
*p = 5; // chyba
```

Pokud tato paměť byla naalokována jinde, přepisujeme si ji, pokud ne, může vyústit v segmentační chybu. 

---
### Memory leak

Paměť nebyla uvolněna.
```c
int *p = malloc(sizeof(int));  
// zapomenuté free(p)
```
