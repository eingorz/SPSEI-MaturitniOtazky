## 7.1 Co to je DDL

**DDL (Data Definition Language)** je část jazyka SQL, která slouží k **definici a správě struktury databáze**.

DDL **nepracuje s daty samotnými**, ale s jejich strukturou.
### 7.1.1 Používá se pro:
- vytváření databází a tabulek
- úpravu jejich struktury
- mazání databázových objektů

## 7.2 **Tvorba databáze a tabulek**

### 7.2.1. Vytvoření databáze

Používá se příkaz:
```mysql
CREATE DATABASE nazev_databaze [CHARACTER SET charset_name]
```

### 7.2.2 Výběr databáze

```mysql
USE Skola;
```

### 7.2.3 Vytvoření tabulky

#### 7.2.3.2 Vytvoření nové tabulky
```mysql
CREATE TABLE nazev_tabulky (  
    sloupec datovy_typ omezeni,  
    ...  
);
```

```mysql
CREATE [TEMPORARY] TABLE [IF NOT EXISTS] osoba (  
    UID INT AUTO_INCREMENT PRIMARY KEY
    NAME VARCHAR(100) NOT NULL,
    USERNAME VARCHAR(32) NOT NULL UNIQUE,
    ID_ROLE INT NOT NULL DEFAULT 1,
    ...,
    
    FOREIGN KEY (ID_ROLE) REFERENCES ROLE(ID)
) [ENGINE engine_name, CHARSET charset_name, ...];
```
- `TEMPORARY` - dočasná tabulka vázaná na aktuálního uživatele, zaniká při ukončení spojení

#### 7.2.3.2 Podle jiné tabulky
```mysql
CREATE TABLE t1 LIKE t2;
```
-  **Zkopíruje se**
	- struktura sloupců (názvy, datové typy)
	- `NOT NULL`, `DEFAULT` hodnoty
	- PRIMARY KEY
	- INDEXY (včetně UNIQUE)
	- AUTO_INCREMENT atribut (ale začína od 1)
	
- **Co se nezkopíruje**
	- data
	- FOREIGN KEY omezení
	- triggery

### 7.3.1 Datové typy (příklady)
- `INT` – celé číslo
- `VARCHAR(n)` – text o max. délce n
- `DATE` – datum
- `FLOAT` – desetinné číslo

### 7.3.2 Omezení (constraints)
- `PRIMARY KEY` – primární klíč (jedinečný identifikátor)
- `NOT NULL` – hodnota nesmí být prázdná
- `UNIQUE` – hodnota musí být jedinečná
- `DEFAULT` - sloupec nabývá defaultní hodnoty, kterou natavíme při tvorbě/úpravě tabulky
- `FOREIGN KEY` – cizí klíč (vazba na jinou tabulku)

## 7.4 Změny tabulek a jejich odstranění
### 7.4.1 Úprava tabulky (ALTER TABLE)
- Používá se pro změnu struktury tabulky.
```mysql
ALTER TABLE Studenti  
[ ADD sloupec dat_typ omezeni [, ADD sloupec2...] ]
[MODIFY sloupec dat_typ omezeni]
[CHANGE stary_nazev novy_nazev dat_typ omezeni]
[DROP COLUMN sloupec]
[RENAME TO novy_nazev_tabulky];

-- alternativa pro rename tabulky
RENAME TABLE Studenti TO NoviStudenti;
```

### 7.4.2 Odstranění tabulky
```mysql
DROP TABLE Studenti;

DROP DATABASE Skola;
-- nebo
DROP SCHEMA Skola;
```

### 7.4.3 Odstranění dat tabulky **(!!!DML!!!)**
- slouží k odstranění všech dat z tabulky
- smaže stávající tabulku a znovuvytvoří (zachová vše, krom dat)

```mysql
TRUNCATE TABLE Studenti;
```
- `DELETE` – maže řádky (lze vrátit v transakci*)
- `TRUNCATE` – rychlé smazání všech dat (nelze vrátit)


\*Transakce
`START TRANSACTION;` - začátek transakce
`COMMIT;` - potvrzení transakce, změny se uloží natrvalo
`ROLLBACK [TO label];` - zrušení transakce, vrátí databázi do stavu před začátkem transakce
`SAVEPOINT label` - tvorba bodu návratu v transakci
