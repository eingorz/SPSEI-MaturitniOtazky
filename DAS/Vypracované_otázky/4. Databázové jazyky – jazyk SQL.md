Databázový jazyk je formální jazyk používaný k **vytváření, správě a dotazování databází**. 
Hlavním cílem je umožnit uživatelům a aplikacím komunikovat s databázovým systémem, ukládat data, upravovat je a získávat z nich potřebné informace.

Nejznámějším databázovým jazykem je **SQL (Structured Query Language)**, který se používá v relačních databázích. 

**SQL umožňuje:**
- definovat strukturu databáze,
- manipulovat s daty (vkládat, upravovat, mazat),
- dotazovat se na data pomocí různých kritérií.

### 4.1 Rozdělení databázových jazyků podle činnosti
Dělí se podle toho, k čemu slouží:
#### 4.1.1 **Jazyk pro definici dat – DDL (Data Definition Language)**
- Slouží k **vytváření a úpravě struktury databáze**: tabulek, pohledů, indexů.
- **Příklady příkazů:**
	- `CREATE TABLE` – vytvoření nové tabulky
	- `ALTER TABLE` – změna struktury tabulky
	- `DROP TABLE` – odstranění tabulky
#### 4.1.2 **Jazyk pro manipulaci s daty – DML (Data Manipulation Language)**
- Slouží k **vkládání, aktualizaci a mazání dat**.
- **Příklady příkazů:**
	- `INSERT INTO` – vložení dat do tabulky
	- `UPDATE` – aktualizace existujících dat
	- `DELETE` – odstranění dat
#### 4.1.3 **Dotazovací jazyk – DQL (Data Query Language)**
- Slouží k **vyhledávání a získávání dat z databáze**.
- **Hlavní příkaz:**
	- `SELECT` – výběr dat podle různých kritérií
#### 4.1.4 **Jazyk pro řízení přístupu – DCL (Data Control Language)**
- Slouží k **udělování a odebírání práv uživatelům** databáze.
- **Příklady příkazů:**
	- `GRANT` – udělení práv uživateli
	- `REVOKE` – odebrání práv uživateli

### 4.2  **Syntaxe jazyků na příkladech**

#### **4.2.1 DDL – definice tabulky**
```SQL
CREATE TABLE studenti (  
    id INT PRIMARY KEY,
    jmeno VARCHAR(50),  
    prijmeni VARCHAR(50),  
    vek INT,  
    studijni_program VARCHAR(100)  
);
```
- `CREATE TABLE` – příkaz pro vytvoření tabulky
- `INT`, `VARCHAR(50)` – datové typy
- `PRIMARY KEY` – označení primárního klíče tabulky

#### **4.2.2 DML – manipulace s daty**

**Vložení dat**
```SQL
-- Musíme dodržet přesné pořadí sloupců v tabulce  
-- Musíme zadat hodnoty pro všechny sloupce (pokud nemají DEFAULT nebo NULL)  
-- Méně bezpečné – změna struktury tabulky může dotaz rozbít  
INSERT INTO studenti VALUES (1, 'Jan', 'Novak', 19, 'Informatika');

-- Nemusíme dodržet pořadí prvků  
-- Můžeme vynechat některé sloupce (pokud mají DEFAULT hodnotu nebo dovolují NULL)
-- Je bezpečnější – nezávisí na pořadí sloupců v tabulce
INSERT INTO studenti (id, jmeno, prijmeni, vek, studijni_program)  
VALUES (1, 'Jan', 'Novak', 19, 'Informatika')
[ON DUPLICATE KEY UPDATE assignment_list];
```

- `INSERT INTO` - příkaz pro vložení do tabulky
- `(id, jmeno, prijmeni, ...)` - sloupce tabulky, do kterých vkládáme data
- `VALUES (1, 'Jan', 'Novak',...)` - hodnoty které vkládáme do tabulky 
- `VALUE` - pro jeden záznam, `VALUES`- pro vícé záznamů
- `[ON DUPLICATE KEY UPDATE]` 
	- Pokud **nedojde ke konfliktu** → řádek se vloží
	- Pokud dojde k **duplicitě (PRIMARY KEY / UNIQUE)** → provede se **UPDATE** existujícího záznamu v tabulce
	- V části `UPDATE` lze použít i hodnoty z vkládaného řádku, např. `VALUES(jmeno)`

**Aktualizace dat**
```SQL
UPDATE studenti  
SET vek = 20, jmeno = 'Pepo'
WHERE id = 1;
```
**Smazání dat**
```SQL
DELETE FROM studenti
WHERE id = 1;
```

#### **4.2.3 DQL – dotazování dat**
```SQL
SELECT jmeno, prijmeni, studijni_program  
FROM studenti  
WHERE vek > 18;
```
- `SELECT` – vybírá sloupce
- `FROM` – určuje tabulku
- `WHERE` – podmínka výběru

### 4.3 **Klíčová slova a datové typy SQL**

#### **4.3.1 Klíčová slova**
- **DDL:** 
	- `CREATE` - tvorba tabulek, pohledů, schémat
	- `ALTER` - úprava tabulek, uživatelů
	- `DROP` - mazání tabulek, pohledů, schémat, constraints
- **DML:** 
	- `INSERT` - vklad záznamů do tabulky
	- `UPDATE` - úprava záznamů v tabulce
	- `DELETE` - mazání záznamů z tabulky
- **DQL:** 
	- `SELECT` - výběr dat
	- `FROM` - určení zdroje dat ze kterého SELECT vybírá
	- `WHERE` - podmínka výběru 
	- `GROUP BY` - seskupení záznamů ve výběru podle atributu, užívá se při použití agregačních funkcí
	- `ORDER BY param [ASC|DESC]` - seřazení záznamů
		- defaultně `ASC` - vzestupně, `DESC` - sestupně
	- `HAVING` - filtruje skupiny vzniké pomocí `GROUP BY`
- **DCL:** 
	- `GRANT` - udělení práv uživatelům nebo rolím
	- `REVOKE` - odstranění práv uživatelům nebo rolím
	- Lze udělovat například práva:
		- **SELECT** – čtení dat
		- **INSERT** – vkládání dat
		- **UPDATE** – aktualizace dat
		- **DELETE** – mazání dat
		- **ALL PRIVILEGES** – všechna práva

#### **4.3.2 Základní datové typy v SQL**
- **Číselné:** `INT`, `SMALLINT`, `BIGINT`, `DECIMAL(M[D])` - {M - počet cifer, D - počet desetinných míst} , `NUMERIC`, `FLOAT`, `REAL`
- **Znaky / text:** `CHAR(n)`, `VARCHAR(n)`, `TEXT`
- **Datum a čas:** `DATE`, `TIME`, `DATETIME`, `TIMESTAMP`
- **Logické:** `BOOLEAN`
