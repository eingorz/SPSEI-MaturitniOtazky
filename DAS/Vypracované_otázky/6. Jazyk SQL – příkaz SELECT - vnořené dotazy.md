## 6.1 Co je SQL a příkaz SELECT

**SQL (Structured Query Language)** je standardizovaný jazyk pro práci s relačními databázemi. Používá se pro:
- dotazování na data
- vkládání, úpravu a mazání dat
- správu databází

### 6.1.1 Základní syntaxe:
```Mysql
SELECT sloupce  
FROM tabulka  
WHERE podmínka;
```

---
## 6.2 Typy spojení tabulek (JOIN)
- Spojení tabulek se používá, když potřebujeme data z více tabulek.

### 6.2.1 INNER JOIN (JOIN)
- Vrací pouze řádky, které mají shodu v obou tabulkách
- Nejčastější typ spojení

```Mysql
SELECT *  
FROM zakaznici  
INNER JOIN objednavky  
ON zakaznici.id = objednavky.zakaznik_id;
```
Vlastnosti:
- odstraní neshodné záznamy
- používá se pro „průnik“

### 6.2.2 LEFT JOIN (LEFT OUTER JOIN)
- Vrací všechny řádky z levé tabulky
- Pokud není shoda, doplní NULL

```Mysql
SELECT *  
FROM zakaznici  
LEFT JOIN objednavky  
ON zakaznici.id = objednavky.zakaznik_id;
```
Vlastnosti:
- zachová všechna data z levé tabulky
- vhodné pro hledání „chybějících“ vztahů

### 6.2.3 RIGHT JOIN (RIGHT OUTER JOIN)
- Opak LEFT JOIN
- Vrací všechny řádky z pravé tabulky

### 6.2.4 FULL JOIN (FULL OUTER JOIN)
- Vrací všechny řádky z obou tabulek
- Pokud není shoda → NULL

### 6.2.5 CROSS JOIN
- Kartézský součin (každý s každým)
```Mysql
SELECT *  
FROM A  
CROSS JOIN B;
```

## 6.3 Jednořádkové vnořené dotazy (skalární poddotazy)
### 6.3.1 Co to je
- Vnořený dotaz (subquery) je SELECT uvnitř jiného dotazu.
- **Skalární poddotaz** vrací právě jednu hodnotu (1 řádek, 1 sloupec).
### 6.3.2 Používá se v
- `WHERE`
- `SELECT`
- `HAVING`

```Mysql
SELECT *  
FROM zamestnanci  
WHERE plat > (  
    SELECT AVG(plat)  
    FROM zamestnanci  
);
```
- vnitřní dotaz vrátí jednu hodnotu (např. 50000)
- vnější dotaz ji použije v podmínce

### 6.3.3 Operátory
- `=`, `>`, `<`, `>=`, `<=`, `<>`

## 6.4 Vnořené dotazy vracející více hodnot
- Tyto poddotazy vrací více řádků → používají se **speciální operátory**.

### 6.4.1 IN
- testuje, zda hodnota patří do množiny
```Mysql
SELECT *  
FROM zamestnanci  
WHERE oddeleni_id IN (  
    SELECT id  
    FROM oddeleni  
    WHERE mesto = 'Praha'  
);
```
- význam: „je v seznamu“

### 6.4.2 ANY (nebo SOME)
- podmínka platí alespoň pro jednu hodnotu
```Mysql
SELECT *  
FROM produkty  
WHERE cena > ANY (  
    SELECT cena  
    FROM produkty  
    WHERE kategorie = 'levne'  
);
```
- význam: větší než alespoň jeden prvek

### 6.4.3 ALL
- podmínka musí platit pro všechny hodnoty
```Mysql
SELECT *  
FROM produkty  
WHERE cena > ALL (  
    SELECT cena  
    FROM produkty  
    WHERE kategorie = 'levne'  
);
```
- význam: větší než všechny → tedy větší než maximum

### 6.4.4 EXISTS
- testuje existenci alespoň jednoho řádku
```Mysql
SELECT *  
FROM zakaznici z  
WHERE EXISTS (  
    SELECT 1
    FROM objednavky o  
    WHERE o.zakaznik_id = z.id  
);
-- Lze použít i 'SELECT * ...', ale v případě EXISTS nepotřebujeme vybírat data z tabulky objednavky, takže se vybírá konstanta a tím dáváme najevo že s daty nepracujme, jen zjišťujeme zda existují
```
- význam: vrací TRUE, pokud poddotaz vrátí alespoň jeden řádek

### 6.4.5 Shrnutí
| **Operátor** | **Význam**                      |
| ------------ | ------------------------------- |
| `IN`         | hodnota je v množině            |
| `ANY/SOME`   | platí pro alespoň jednu hodnotu |
| `ALL`        | platí pro všechny hodnoty       |
| `EXISTS`     | existuje alespoň jeden řádek    |
