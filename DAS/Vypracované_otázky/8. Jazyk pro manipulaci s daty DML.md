## 8.1 Co je DML

**DML (Data Manipulation Language)** je část jazyka SQL, která slouží k **práci s daty v databázi**.  

Na rozdíl od DDL (které pracuje se strukturou databáze) DML pracuje **přímo s obsahem tabulek**.

### 8.1.1 Používá se k tomu, abychom mohli:
- vkládat, měnit, mazat, získávat

## 8.2 Postup práce s daty v databázi
### 8.2.1 Přidávání dat `INSERT`
```mysql
INSERT INTO studenti (id, jmeno, prijmeni, vek, studijni_program)  
VALUES (1, 'Jan', 'Novak', 19, 'Informatika')
[ON DUPLICATE KEY UPDATE assignment_list];
```
- `INSERT INTO` - příkaz pro vložení do tabulky
- `(id, jmeno, prijmeni, ...)` - sloupce tabulky, do kterých vkládáme data
- `VALUES (1, 'Jan', 'Novak',...)` - hodnoty které vkládáme do tabulky 
- `VALUE` - pro jeden záznam, `VALUES`- pro vícé záznamů
- `[ON DUPLICATE KEY UPDATE]` 
	- Pokud **nedojde ke konfliktu** → řádek se vloží
	- Pokud dojde k **duplicitě (PRIMARY KEY / UNIQUE)** → provede se **UPDATE** existujícího záznamu v tabulce
	- V části `UPDATE` lze použít i hodnoty z vkládaného řádku, např. `VALUES(jmeno)`

```mysql
-- Musíme dodržet přesné pořadí sloupců v tabulce  
INSERT INTO studenti VALUES (1, 'Jan', 'Novak', 19, 'Informatika')

-- Nemusíme dodržet pořadí prvků  
INSERT INTO studenti (id, jmeno, prijmeni, vek, studijni_program)  
VALUES (1, 'Jan', 'Novak', 19, 'Informatika')
```

### 8.2.2 Úprava dat `UPDATE`
- mění exisitující data
```mysql
UPDATE [IGNORE] zakaznici
SET vek = 26
[WHERE jmeno = 'Jan' AND prijmeni = 'Novák']
[ORDER BY ...]
[LIMIT row_count]
-- Pokud vynecháme `WHERE`, změní se všechny řádky [v limitu]
```
- `IGNORE` - pokud dojde k chybě při vkládání dat, data se nevloží, ignoruje chybu a pokračuje dál

### 8.2.3 Odstraňování dat `DELETE`
```mysql
DELETE FROM zakaznici
[WHERE vek < 18]
[ORDER BY ...]
[LIMIT row_count]
-- Pokud vynecháme `WHERE`, smažou se všechny řádky [v limitu]
```

```mysql
TRUNCATE TABLE Studenti;
```
- slouží k odstranění všech dat z tabulky
- smaže stávající tabulku a znovuvytvoří (zachová vše, krom dat)
- rychlé smazání všech dat (nelze vrátit v transakci*, protože se jedná o samostatný commit)

\*Transakce
`START TRANSACTION;` - začátek transakce
`COMMIT;` - potvrzení transakce, změny se uloží natrvalo
`ROLLBACK [TO label];` - zrušení transakce, vrátí databázi do stavu před začátkem transakce
`SAVEPOINT label` - tvorba bodu návratu v transakci

### 8.2.4 Získávání dat `SELECT
[[5. Jazyk SQL – příkaz SELECT]]

```mysql
SELECT sloupce  
FROM tabulka  
[WHERE podmínka]  
[GROUP BY sloupce]  
[HAVING podmínka]  
[ORDER BY sloupce];
```
#### 8.2.5 Povinné části:
- **SELECT** – určuje, které sloupce chceme zobrazit
- **FROM** – určuje tabulku (nebo tabulky), ze kterých data bereme

#### 8.2.6 Nepovinné části:
- **WHERE** – filtruje řádky (podmínka výběru)
- **GROUP BY** – seskupuje řádky do skupin
- **HAVING** – filtruje skupiny (na rozdíl od WHERE)
- **ORDER BY** – řazení výsledků
- **DISTINCT** – odstranění duplicit
- **JOIN** – spojování tabulek
