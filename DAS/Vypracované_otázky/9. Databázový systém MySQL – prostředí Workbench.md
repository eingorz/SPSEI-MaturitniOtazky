## 9.1 Co to je MySQL Workbench
- Grafické nástrojové prostředí (GUI), které slouží ke správě databázového systému MySQL.

- Umožňuje vytvářet databáze, navrhovat jejich strukturu, psát SQL dotazy, spravovat uživatele i sledovat výkon serveru.

---
## 9.2 Pracovní prostředí MySQL Workbench

### 9.2.1 Home (Domovská obrazovka)
- Přehled uložených připojení k databázím
- Možnost vytvořit nové připojení

### 9.2.2 SQL Editor
- Hlavní pracovní plocha
- Slouží k psaní a spouštění SQL příkazů (např. `SELECT`,` INSERT`,` UPDATE`)
- Obsahuje:
    - editor dotazů
    - tlačítka pro spuštění dotazů
    - výpis výsledků

### 9.2.3 Navigator (Navigační panel)
- Levý panel
- Zobrazuje strukturu databáze:
    - databáze (schemas)
    - tabulky
    - pohledy
    - procedury

### 9.2.4 Output (Výstupní panel)
- Informace o provedených operacích (čas, počet ovlivněných řádků)
- Chybové hlášky
- Logy

### 9.2.5 Administration
- Správa serveru:
	- uživatelé a oprávnění
	- zálohování databází
	- monitoring výkonu

## 9.3 Vlastnosti objektů databázového systému a práce s nimi

### 9.3.1 Databáze (Schema)
- Logický celek obsahující tabulky a další objekty (procedury, pohledy, ...)
- Operace:
    - vytvoření: `CREATE DATABASE`
	- smazání: `DROP DATABASE`

### 9.3.2 Tabulky (Entita)
- Základní objekt pro ukládání dat
- Skládají se z:
    - sloupců (atributů)
    - řádků (záznamů)
#### 9.3.2.1 Vlastnosti:
[[7. Jazyk pro definici dat DDL]]]
- datové typy (INT, VARCHAR, DATE…)
- primární klíč (PRIMARY KEY)
- cizí klíč (FOREIGN KEY)
- omezení (NOT NULL, UNIQUE)

#### 9.3.2.2 Práce:
- [[8. Jazyk pro manipulaci s daty DML]]
- vytvoření tabulky (Table Editor)
- úprava struktury
- vkládání dat (INSERT)
- mazání (DELETE)

### 9.3.3 Indexy
- Slouží ke zrychlení vyhledávání dat v příkazu `SELECT`
- Nevýhoda: zpomalují zápis dat

### 9.3.4 Relace
- Vazby mezi tabulkami
- Typy:
    - 1:1
    - 1:N
    - N:M (pomocí spojovací tabulky)

### 9.3.5 Uživatelé a oprávnění
- Správa přístupů:
    - `SELECT`, `INSERT`, `UPDATE`, `DELETE`
- Nastavení práv uživatelů v sekci Administration

### 9.3.6 Práce v MySQL Workbench
- návrh databáze pomocí EER diagramu (vizuální modelování)
- úpravy pomocí editorů (tabulka, sloupce)
- spouštění SQL skriptů
- import/export dat

## 9.4 Úloha pohledů (Views) v databázových systémech

### 9.4.1 Co je pohled
- Virtuální tabulka založená na SQL dotazu
- Neobsahuje vlastní data – pouze zobrazuje data z jiných tabulek

### 9.4.2 Vlastnosti pohledů
- definovány pomocí SELECT
- ukládají pouze dotaz, ne data
- mohou kombinovat více tabulek (JOIN)

### 9.4.3 Výhody pohledů
- **Zjednodušení dotazů**
    - složité dotazy se uloží jako pohled
- **Bezpečnost**
    - uživatel vidí jen vybraná data
- **Abstrakce**
    - oddělení logiky od fyzické struktury databáze

```mysql
CREATE VIEW aktivni_zakaznici AS
SELECT jmeno, prijmeni
FROM zakaznici
WHERE aktivni = 1;
```
### 9.4.4 Vkládání dat přes pohled
- Vkládání (`INSERT`) je možné jen pokud je pohled jednoduchý:
-  Podmínky:
	- pohled je založen na **jedné tabulce**
	- obsahuje všechny **povinné sloupce** (NOT NULL bez defaultu)
	- neobsahuje:
	    - JOIN
	    - GROUP BY
	    - agregační funkce (SUM, COUNT…)
	    - DISTINCT
